# System — Story Memory

## Purpose

Tracks narrative-level similarity across generated stories.

Prevents repetition of:

- core scenario
- narrative premise
- character situation
- emotional arc

---

## Memory Entry Format

### Story ID:

Core Scenario:
- (e.g. alone in house, being watched, followed, etc.)

Setting:
- (house, car, workplace, outdoors, etc.)

Primary Fear:
- (intrusion, surveillance, isolation, deception, etc.)

Narrative Arc:
- (slow realization, pursuit, entrapment, etc.)

Key Twist:
- (what changes the story)

---

## Repetition Rules

A new story MUST NOT:

- reuse the same core scenario + setting
- reuse the same narrative arc consecutively
- reuse identical fear dynamics

---

## Soft Variation Rules

The system SHOULD:

- rotate environments
- vary social context (alone vs multiple people)
- vary discovery method (sound, visual, interaction)

---

## Compression Summary

Track → Compare → Vary scenario

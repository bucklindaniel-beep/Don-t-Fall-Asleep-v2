# Pattern → Framework Mapping

## Purpose
Defines how extracted narrative patterns map to system frameworks.

Ensures:
- consistent system usage
- no overlap between frameworks
- predictable generation behavior

---

## Entry Patterns → 01_horror_escalation_model

Entry patterns control:
- how the story begins
- what introduces the anomaly

Examples:
- direct contact initiation
- distant observation
- environmental signal entry

---

## Escalation Patterns → 05_physical_escalation_model

Escalation patterns control:
- how threat increases
- how proximity and control evolve

Examples:
- repeated contact
- spatial constriction
- boundary testing
- parallel tracking

---

## Interaction Patterns → 03_psychological_breakdown_model

Interaction patterns control:
- how the entity behaves
- how the narrator is influenced

Examples:
- false assistance
- recruitment framing
- passive observation

---

## Environment Patterns → 02_environmental_escalation_model

Environment patterns control:
- how the setting limits or exposes the narrator
- how space contributes to threat

Examples:
- isolation amplification
- interior compromise
- encirclement

---

## Resolution Patterns → 09_retention_mechanics

Resolution patterns control:
- how the story ends
- what is withheld vs revealed

Examples:
- escape without explanation
- authority interruption
- behavioral change

---

## Scene Execution → 06_scene_construction_system

All structured scenes must be built using:
- escalation stage alignment
- pattern-driven behavior
- environment constraints

---

## Tension Control → 08_tension_pacing_system

Controls:
- sentence rhythm
- escalation timing
- information spacing

---

## System Orchestration → 07_pattern_and_rule_system

Controls:
- which patterns are selected
- how patterns are combined
- enforcement of non-repetition

# System — Continuity Lock

## Purpose
Maintains visual consistency across all shots within a scene and across scenes.

Ensures:
- characters remain visually consistent
- environments do not reset or shift unexpectedly
- objects persist and evolve logically
- lighting and atmosphere progress naturally

---

## Core Principle

Each shot inherits the state of the previous shot.

No shot exists in isolation.

---

## Continuity Layers

### 1. Character Continuity
- appearance (face, hair, clothing)
- posture and condition
- injuries or changes persist

---

### 2. Environment Continuity
- room layout
- object placement
- spatial relationships
- environmental damage or changes

---

### 3. Object Continuity
- introduced objects must persist
- objects cannot disappear without cause
- object position changes must be logical

---

### 4. Lighting Continuity
- light sources remain consistent
- changes must be gradual or explained
- intensity and direction must not reset

---

### 5. State Progression
Each shot may:
- add detail
- introduce change
- escalate condition

But MUST NOT:
- reset state
- contradict prior shot

---

## Continuity Lock Rule

Each shot MUST include:

- reference to previous state
- only incremental changes

---

## Allowed Changes

Changes must be:

- additive (new element appears)
- progressive (existing element changes)
- reactive (result of prior action)

---

## Forbidden Changes

- character redesign between shots
- environment layout shift without transition
- object removal without explanation
- lighting reset
- tone reset

---

## Failure Conditions

Continuity FAILS if:

- two adjacent shots contradict visually
- environment resets between shots
- character appearance changes
- introduced elements disappear without cause

---

## Compression Summary

Previous State → Apply Change → New State → Next Shot

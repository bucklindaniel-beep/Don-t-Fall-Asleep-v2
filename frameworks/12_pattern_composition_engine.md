# Pattern Composition Engine

## Metadata
- Type: Framework
- Domain: Pattern Selection / Creativity / Story Planning
- Status: active
- Priority: high

---

## Purpose
Assemble compatible mechanics before generation so stories are original, controlled, and non-repetitive.

---

## Hierarchy
1. Extract mechanics.
2. Score candidate patterns.
3. Compose a small compatible stack.
4. Generate original output.
5. Route only successful reusable patterns to memory.

---

## Pattern Stack
A stack may include:
- organic hook type
- foundation method
- story beat flow
- escalation pattern
- curiosity loop
- pattern break
- hidden setup/payoff
- resolution pattern

Use fewer strong mechanics over many weak mechanics.

---

## Compatibility Rules
Patterns must fit:
- narrator voice
- horror tone
- escalation type
- pacing model
- resolution style
- YouTube audience expectations without clickbait tone

Avoid combinations that create:
- aggressive hook tone in slow-burn stories
- constant intensity with no contrast
- too many twists without setup
- repeated structure from recent story memory

---

## Diversity Rule
Check `/analysis/patterns/story_uniqueness_memory.md` before reuse.
Vary hook type, narrator behavior, escalation path, and resolution pattern across runs.

---

## Output Discipline
Do not expose the pattern stack in the creative output unless diagnostics are requested.

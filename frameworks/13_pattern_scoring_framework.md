# Pattern Scoring Framework

## Metadata
- Type: Framework
- Domain: Pattern Evaluation / Promotion / Memory Hygiene
- Status: active
- Priority: high

---

## Purpose
Score candidate patterns before reuse or promotion so memory improves quality instead of accumulating noise.

---

## Score Dimensions
Rate 1–5 only when evidence exists.

- **Engagement value:** creates curiosity, tension, payoff, or emotional movement
- **Reuse value:** works across multiple original stories
- **Originality value:** avoids generic niche repetition
- **Tone fit:** preserves grounded horror voice
- **Compatibility:** works with selected hook, beat flow, escalation, and resolution
- **Evidence strength:** supported by transcript analysis, completed run, user feedback, or performance data

---

## Promotion Rule
Promote only when the pattern has:
- clear reusable function
- clear use case
- clear non-use case
- low copying risk
- medium or high total value

---

## Deprioritize
Deprioritize patterns that are:
- source-specific
- decorative only
- clickbait-dependent
- likely to homogenize outputs
- incompatible with grounded horror tone

---

## Performance Data Rule
When YouTube metrics become available, increase evidence strength only for patterns linked to retention, CTR, comments, completion, or repeatable audience response.
Until then, treat scoring as editorial confidence, not proof.

---

## Memory Rule
Memory entries must explain:
- why the pattern works
- when to use it
- when not to use it
- whether it should be reused, tested, or held

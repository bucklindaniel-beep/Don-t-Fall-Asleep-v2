## Purpose
Central source of truth for prose and narration quality. Other files should reference this file instead of duplicating these rules.

---

## Rules

### Behavior Over Labels
Show traits through observable behavior, not abstract descriptors.

### Descriptive Density
Description must serve plot, character state, perception, or tension. Avoid stacked descriptors and reflexive triplets unless each word adds distinct meaning.

### Inner Monologue Value
Internal thoughts must reveal character, contradiction, tension, or curiosity. Avoid procedural chains.

### First-Person Variation
Reduce repeated “I” openings and step-by-step reporting. Compress neutral actions into meaningful beats.

### Rhetorical Pattern Control
Use frames like “I know how this sounds” or “I want you to understand” only when context earns them.

### Implicit Time Progression
Use contextual time cues over clock precision unless exact time is plot-critical.

### Em Dash Control
Use em dashes sparingly. Do not let them become the default rhythm.

### Thought Impact
Key thoughts should create curiosity, shift understanding, or deepen tension.

### Performance Awareness
Strong prose is not enough. Openings, turns, and endings must support retention without becoming clickbait.
